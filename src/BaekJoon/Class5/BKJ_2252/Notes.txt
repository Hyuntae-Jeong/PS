풀이 접근
- LinkedList를 사용해서 줄을 세우고, 2차원 배열을 이용해서 두 학생간의 관계를 저장하자 (relation[M+1][M+1])
- boolean 배열로 학생이 줄을 섰는지 여부를 저장하자 (isStanding[N])
- linkedList는 양방향이고, pool을 사용해서 O(1) 접근이 가능하다

풀이 방법

입력:
- 두 학생의 관계 "a b" 를 relation[a][b], relation[b][a]에 저장한다
    - relation[a][b] = 1  => b는 a 뒤에 서야 한다
    - relation[a][b] = -1 => b는 a 앞에 서야 한다
    - relation[a][b] = 0   => a와 b는 아무 관계가 없다
- "a b"가 입력될 경우
    - relation[a][b] = 1
    - relation[b][a] = -1


해결:
- 1번 학생(a)부터 relation 배열과 isStanding 배열을 보면서 linkedList를 채워 나간다
- 상대방(b)이 isStanding이 false이면서 relation이 1 또는 -1 이면 관계를 적당히 보면서 줄을 세운다
- 아무 관계가 없을 때에는 그냥 연결시키지 않는다

비교 로직:
- b가 a보다 앞에 서야하는데, 이미 b 앞에 누군가 서있을 경우 (a.front != null)
    - a.front와 관계를 비교한다.
        - 앞에 서야할 경우 다시 그 앞에 사람과 비교하고
        - 뒤에 서야할 경우 그 자리에 줄을 스고
        - 관계 없을 경우에는 또 그 앞사람이 있을 경우 앞 사람과 비교를 시작한다

- b가 a보다 뒤에 서야하는데, 이미 b 뒤에 누군가 서있을 경우 (a.back != null)
    - a.back과 관계를 비교한다
        - 뒤에 서야할 경우 다시 그 뒤 사람과 비교하고
        - 앞에 서야할 경우 그 자리에 줄을 스고
        - 관계가 없을 경우에는 또 그 뒷사람과 비교를 시작한다 (or 그냥 그 자리에 선다)

위상정렬 방법에는 BFS 방식과 DFS 방식이 있다

1) BFS 방식
앞에 와야하는 노드 수 리스트(incoming)와 큐 자료구조 사용
<steps>
1. incoming[] 값이 0인 노드를 queue에 넣고 BFS 탐색 시작
2. queue에서 노드를 빼서 결과 리스트에 추가하고, 해당 노드에 연결된 노드들의 incoming[] 값에 -1을 해줌.
2-1. 만약 0이 된다면 queue에 넣어줌
3. 큐가 빌 때까지 이 과정 반복

2) DFS 방식
재귀 호출을 활용한 깊이 탐색
아무 노드나 찍어서 깊이 탐색하고 노드의 끝에 도달하면 (더이상 방문할 노드가 없는 노드) 결과 리스트에 추가하고 리턴
<steps>
1. 아직 방문하지 않은 노드 DFS 호출
2. 해당 노드에서 더 방문할 노드가 있으면 DFS 호출
3. 방문한 노드가 더이상 탐색할 노드가 없으면 방문으로 표시하고 결과 리스트에 추가하고 리턴함
4. DFS 호출 후 아직 방문하지 않은 노드가 있다면 해당 노드에서 DFS 또 시작


3) BFS vs DFS
BFS = 앞에서부터 구해나가는 방식
DFS = 뒤에서부터 구해나가는 방식

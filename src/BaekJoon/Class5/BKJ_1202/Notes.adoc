== 문제 접근
1. 보석의 가격으로 내림차순 정렬을 한다
2. 각 보석이 들어갈 수 있는 가장 작은 가방을 이진탐색으로 찾는다
3. 담을 가방이 없으면 담지 않는다

== 고민 포인트
이진탐색으로 가방을 찾은 뒤에 가방 리스트를 삭제하기??
이진탐색은 인덱스 기반으로 이루어지는데, 리스트에서 삭제하면 인덱스 관리가 어려워짐.
배열에서 해당 가방을 사용할 수 있는지 여부도 표시해야 할 듯

== 이진탐색 과정
1. L R 포인터가 있을 때, (L + R) / 2의 가방을 탐색한다.
2. 일단 가방 사용가능 여부를 확인하고 만약 사용이 가능하면서 지금까지 찾은 가방 중 최소 무게일때에는
해당 값과 인덱스를 기억해둔다.
3. 가방 사용 가능 여부와 관련 없이
해당 가방이 담을 수 있는 무게가 보석의 무게보다 작으면 그 왼쪽 가방들은 볼 필요 없기 때문에 L을 (L + R) / 2 + 1 로 설정한다
해당 가방이 담을 수 있는 무게가 보석의 무게보다 크면 R를 (L + R) / 2로 설정한다.
해당 가방이 담을 수 있는 무게가 보석의 무게와 같다면 해당 가방을 사용하고 즉시 종료한다.
4. L와 R가 만났는데, 둘다 사용한 가방이라면, 해당 인덱스부터 오른쪽으로 순회하면서 담을 수 있는 가장 작은 가방을 찾는다.

== 어떤 구조체를 사용할 것인지
후보 1: ArrayList
ArrayList도 인덱스로 접근이 가능하다. 하지만 list.remove를 하면 뒤에 항목들의 인덱스를 앞당겨야해서 O(N)의 시간 복잡도가 발생한다. 그렇기 때문에 사용 불가!

후보 2: 고정 크기 리스트
이 경우에는 이미 사용한 가방도 리스트에 남기 때문에 이를 고려해서 개발을 해야한다.

후보 3: 링크드 리스트 Pool 관리
고정 크기 리스트로 담을 수 있는 가장 작은 가방 찾고, 해당 가방이 이미 사용중이라면
링크드 리스트를 타고 다음 담을 수 있는 가방을 O(1)만에 찾는다

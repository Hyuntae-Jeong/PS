이 문제는 어떤 지점에서 다시 위치로 돌아왔을 때 비용이 음수인 경우를 판단하는 문제이다

다시 돌아왔을때 음수가 되는 경우는 다음과 같다
- 최단거리 구했을 때 음수사이클이 탐지되는 경우

클러스터가 나뉘어져서 음수 사이클 없는 클러스터만 탐지되는 경우 존재할 가능성 있음
-> 일단 한 지점에서 음수 사이클 탐색 후 없으면 아직 도달하지 않은 지점에서 사이클 탐색 시작
-> 음수 사이클 찾을 때까지 반복하거나 모든 노드 다 탐색하면 종료

음수 사이클은 벨만 포드 알고리즘으로 탐색 가능
벨만 포드는 모든 간선을 정점-1 만큼 반복하기 때문에 입력으로 주어지는 순서대로 배열에 넣고 순차적으로 탐색하면 될 듯

손코딩으로 간선 입력 순서대로 벨만 포드 알고리즘 수행 시에도 문제 풀이되는지 확인하기 => 확인됨, 가능함

한번의 벨만 포드 알고리즘 수행으로 음의 사이클을 탐지하는 것이 목적이라면 슈퍼 소스(Super source) 방식이 존재함
그것은 바로 새로운 정점을 추가하고 그 정점으로부터 나머지 모든 정점에 간선 0을 연결하는 것
그렇게 되면 간선이 추가 됐으므로 v번 탐색하고 v+1 번째 dist 값이 줄어드면 음의 사이클이 존재하는 것임

하지만 굳이 정점 1개와 간선 v개를 추가하지 않고 dist를 모두 0으로 초기화하고 v-1 대신 v번 탐색하면 됨
왜냐하면 어차피 슈퍼 소스 방식대로 벨만 포드를 1회 수행하면 모든 dist값이 0으로 바뀌기 때문이다.
불필요한 간선과 정점 추가로 벨만포드 수행 연산수를 늘리지 않고 그냥 dist값 0으로 초기화 한 후에 v번 탐색하고 v+1번째에서 음의 사이클 탐지하면 된다

dist값을 모두 0으로 초기화하는 이유는 우리는 양의 최단거리는 상관이 없기 때문이다
음의 사이클을 탐지하는 것이 목적이므로 길이가 계속 줄어드는 지만 관심사이기 때문에 0으로 초기화한다
